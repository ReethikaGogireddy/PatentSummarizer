# Patent Summarizer

End-to-end pipeline that clusters, summarizes and visualizes large patent corpora
using **TF-IDF · LDA · Sentence-BERT · K-Means · Hierarchical Clustering · t-SNE**.

**Group 22 · Project 9 · CSE 573 Spring 2026 · Arizona State University**

---

## Repository Structure

```
PatentSummarizer/
├── CODE/                          <- All pipeline source code
│   ├── main.py                    <- Pipeline entry point
│   ├── ingestion.py               <- Data loading (synthetic + PatentsView API)
│   ├── preprocessing.py           <- Text cleaning & tokenization
│   ├── features.py                <- TF-IDF/LSA, LDA, Sentence-BERT embeddings
│   ├── clustering.py              <- K-Means++, Hierarchical, metrics, optimal-k
│   ├── summarization.py           <- Extractive summaries, ROUGE, keyword coverage
│   ├── visualization.py           <- t-SNE, PCA, topic bars, heatmaps, charts
│   └── report.py                  <- Self-contained HTML report generator
│
├── DATA/                          <- Sample datasets
│   └── sample_patents.csv         <- 48 synthetic patents across 6 tech domains
│
├── EVALUATIONS/                   <- Pipeline outputs (generated by running main.py)
│   ├── patent_analysis_report.html  <- Full interactive HTML report
│   ├── patent_clusters.csv          <- Patent-to-cluster assignments
│   └── *.png                        <- 15 visualization figures
│
├── Dockerfile                     <- Containerized run environment
├── docker-compose.yml             <- Three pre-configured run modes
└── requirements.txt               <- Python dependencies
```

## Quickstart

### Docker

```bash
# 1. Clone and enter the repo
git clone <repo-url>
cd PatentSummarizer

# 2. Build the image (first time, ~5 min)
docker compose build

# 3. Run the pipeline
docker compose up patent-pipeline
```

Results appear in `./EVALUATIONS/`.

### Local Python

```bash
# Run from the repo root (not from inside CODE/)
py -3.11 CODE/main.py --source synthetic --n 48 --k 6 --topics 6
```

## All Run Options

### Docker Compose

```bash
# Synthetic corpus (offline, 48 patents — default)
docker compose up patent-pipeline

# Real patents from PatentsView API (requires internet)
docker compose up patent-pipeline-real

# Quick smoke-test (no charts, no report)
docker compose up patent-pipeline-test
```

### Local Python

```bash
# Synthetic corpus (fast, offline)
py -3.11 CODE/main.py --source synthetic --n 48 --k 6 --topics 6

# Real patents via PatentsView API, 500 docs, 8 clusters
py -3.11 CODE/main.py --source patentsview --n 500 --k 8 --topics 8

# Skip visualizations (faster)
py -3.11 CODE/main.py --no-viz --no-report
```

## CLI Arguments

| Argument      | Default     | Description                                  |
| ------------- | ----------- | -------------------------------------------- |
| `--source`    | `synthetic` | `synthetic` (offline) or `patentsview` (API) |
| `--n`         | `48`        | Number of patents to load                    |
| `--k`         | `6`         | Number of clusters                           |
| `--topics`    | `6`         | Number of LDA topics                         |
| `--no-viz`    | off         | Skip all chart generation                    |
| `--no-report` | off         | Skip HTML report                             |

def load_csv(path: str) -> pd.DataFrame:

## Pipeline Architecture

```
Raw Patents (USPTO / PatentsView / synthetic)
    |
    v
[CODE/ingestion.py]      Load & structure patent records
    |
    v
[CODE/preprocessing.py]  Clean -> Tokenize -> Remove stopwords
    |
    +-----------------------------+
    v                             v
[TF-IDF + LSA]           [Sentence-BERT]
[LDA Topic Model]         dense 384-dim embeddings
    |                             |
    +------------+----------------+
                 v
    [CODE/clustering.py]
    K-Means++ | Hierarchical (Ward)
    Silhouette | Davies-Bouldin | Stability ARI
                 |
       +---------+---------+
       v         v         v
  [Metrics]  [CODE/summarization.py]  [CODE/visualization.py]
  Silhouette  Keywords   t-SNE / PCA
  DB Index    ROUGE-1/2/L  Topic bars
  Stability   Coverage     Heatmap
              Centroid Sim
                 |
                 v
    [CODE/report.py] -> EVALUATIONS/patent_analysis_report.html
```

## Troubleshooting

**PatentsView returns no results**
The API occasionally has downtime. Run with `--source synthetic` to bypass.

**Permission denied on EVALUATIONS/ folder (Linux)**
```bash
mkdir -p EVALUATIONS
chmod 777 EVALUATIONS
docker compose up patent-pipeline
```

**Docker Desktop memory (Windows)**
In Docker Desktop -> Settings -> Resources, allocate at least **4 GB RAM**
(Sentence-BERT requires ~400 MB).
